How To Use This Tool

Follow these 4 steps. You can load the example brief if you want to see a complete session first.

1

Describe the feature

Fill in the Feature Brief with the story, summary, risks, and optional personas or platforms.

2

Generate a session pack

Click Generate Session Pack to get test ideas, risky probes, and a coverage map.

3

Test and save bug drafts

Use the Live Session and Bug Drafting sections while testing. Save one or more bug drafts as you go.

4

Save or export

Save the full session to history, export session notes, or export a Jira paste-ready bug draft.

What each section is for
  • Feature Brief: the input about what changed.
  • Session Pack: the generated ideas for what to test.
  • Live Session: your notes while exploring.
  • Bug Drafting: structured defect notes you can save and export.
  • Saved Sessions: your reusable history.

What makes a good brief?
  • State the user goal and the risky outcome if the feature fails.
  • Mention roles, channels, environments, or external systems involved.
  • List recent bugs, constraints, or areas that changed late.

        • Note what you skipped on purpose so the coverage gap is explicit.

        Step 3b. Save one or more defects while the context is still fresh.

        Bug drafting tips
        • Write the title as observed behavior plus affected area.
        • Use expected vs actual to show user impact, not just technical mismatch.
        • Keep steps short enough that another QA can replay them quickly.

        Best Use

        • Start with a risky user workflow, not a generic ticket title.

        • Reuse saved sessions before testing similar features again.

        • Jira paste-format export for defect filing.
